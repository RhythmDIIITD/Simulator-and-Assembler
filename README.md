
# Simple RISC-V Assembler and Simulator

A Python-based simulator for the **RISC-V 32-bit Instruction Set Architecture (ISA)**.  
This simulator reads a binary instruction file (generated by the Simple Assembler) and produces a **trace output file** containing register values after each instruction execution and the memory state after the Virtual Halt instruction.  

It also performs **error detection**, printing any invalid instruction or runtime error to the terminal with the corresponding line number.

---

## Features

- Executes **RISC-V 32-bit binary instructions**.
- Tracks **register values (x0–x31)** after every instruction.
- Maintains a **32 × 32-bit memory** model and prints memory contents after the Virtual Halt.
- Provides **error messages** for invalid instructions or memory access issues.
- Generates a single output file combining **instruction trace** and **memory snapshot**.
- Compatible with binaries generated from the **Simple Assembler**.

---

## Tech Stack

- **Language:** Python 3  
- **Concepts:** RISC-V ISA, Instruction Execution, Program Counter, Virtual Registers, Memory Management, File I/O

## Project Structure

Run the simulator:

python simulator.py <input_binary_file> <output_trace_file>


<input_binary_file> → Path to the binary instruction file generated by the assembler

<output_trace_file> → Path to save the trace output

Example:

python simulator.py test_binaries/test1.bin output/test1_trace.txt
Output Format

After each instruction execution:

{Program Counter} {x0} {x1} {x2} ... {x31}


After Virtual Halt (memory snapshot):

32-bit binary data (memory address 0x0000_0000)
32-bit binary data
...
32-bit binary data (last memory address)


All stats (instruction trace + memory snapshot) are stored in a single output file.

Example

If the input binary program contains a Virtual Halt at the end, the output file will first list register values after each instruction, followed by memory values starting from address 0x0000_0000:

Simple RISC-V Assembler
A Python-based assembler for the RISC-V 32-bit Instruction Set Architecture (ISA).  
This assembler reads an assembly program as an input text file and generates the corresponding binary instructions as an output text file. It also performs syntax and semantic checks, providing error notifications with line numbers when necessary.
---
Features
Converts RISC-V 32-bit assembly instructions into 32-bit binary instructions.
Supports labels for branch and jump instructions.
Handles empty lines, instructions, and labels correctly.
Performs error detection including:
Typos in instruction or register names.
Illegal immediate values (out of bounds).
Missing Virtual Halt instruction.
Virtual Halt not being the last instruction.
Generates a binary output file when code is error-free.
Provides error notifications (stdout) with line numbers for any invalid instructions.
Includes 10 test cases with expected outputs for validation.
Project Structure
Run the assembler:
python assembler.py <input_assembly_file> <output_binary_file>

<input_assembly_file> → Path to the input .asm file
<output_binary_file> → Path for saving the output .bin file
Example:
python assembler.py test_cases/test1.asm output/test1.bin

Memory Layout
Program Memory: 256 bytes (64 × 32-bit instructions), addresses 0x0000_0000 → 0x0000_00FF
Stack Memory: 128 bytes (32 × 32-bit registers), grows downwards, addresses 0x0000_0100 → 0x0000_017F
Data Memory: 128 bytes (32 × 32-bit words), using first 32 locations, addresses 0x0001_0000 → 0x0001_007F
